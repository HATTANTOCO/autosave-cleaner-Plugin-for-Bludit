# autosave-cleaner-Plugin-for-Bludit
When the administrator logs in and the admin panel loads, all page data beginning with 'autosave-' will be deleted in bulk.


# 要件・仕様など
- 本プラグインは、管理者権限で管理画面にログインしたタイミングで autosave- で始まるページデータを一括削除します。
- 動作確認バージョンは、Bludit 3.18.4。

そもそも、Bluditは、オートセーブがむやみに生成される（プレビューしただけで生成されたりもする）ので、気がつくとコンテンツ管理の「自動保存」内に山のようにリビジョンの違うオートセーブページが出来上がっている。
最終的に、それらのほとんどが必要ないページになってしまうのだけれど、削除しようにも、「自動保存」と「Pages（公開）」を行きつ戻りつ、一つ一つマメに削除しなければならず、うんざりしてしまう。
「自動保存」機能自体を止めてしまうのも考えたけれど、「自動保存」は、編集途中にうっかり保存しわすれたときなどには、きっと助かる機能なのかもしれない、というわけで、作ってみたのが本プラグインです。<br>
共通したニーズの方がもしいらしたら、使って見てくださいませ。


# インストール
- CodeボタンあるいはReleasesからダウンロードした圧縮ファイルを解凍後、生成されたフォルダ名 autosave-cleaner-Plugin-for-Bludit［-masterやバージョン名］内の autosave-cleaner フォルダを /bl-plugins/autosave-cleaner/ として配置します。
- Bluditの管理画面のプラグイン（設定）に入り、表示されている autosave-cleaner プラグイン を有効化して下さい。


# ご注意
- 本プラグインは、プラグインを有効化し管理画面をログアウトした後（セッションが切れても同様）、再度、管理者権限で管理画面にログインした時点で、全ての自動保存データを削除します。
- 削除したくない「自動保存」データは、ログアウトする前に必ず「公開」あるいは「下書き」のステータスで「保存」設定を行うなどしてください。
